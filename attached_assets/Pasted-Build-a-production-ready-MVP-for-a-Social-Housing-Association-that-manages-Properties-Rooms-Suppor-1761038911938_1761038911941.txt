Build a production-ready MVP for a Social Housing Association that manages Properties, Rooms, Support-Worker assignments, Tenant onboarding, and secure Document uploads (Proof of ID/Income) with verification and basic reports, using a Replit monorepo where the API is Node.js (Express + Prisma + Zod + JWT with httpOnly cookies) and the web app is React (Vite + Tailwind + React Query + React Hook Form). Use Neon Postgres (free tier) as the database and store document files inside Postgres as BYTEA (no S3 or paid storage). Implement a DocumentBlob table linked to Document; on upload, validate file type (PDF/JPG/PNG) and size (env MAX_UPLOAD_MB, default 10), compute SHA-256 for deduplication, and encrypt bytes using AES-256-GCM with a key from Replit Secrets before writing to DB. Provide streaming download endpoints that require short-lived signed tokens (JWT containing documentId, userId, and exp), and re-enforce RBAC so Support Workers only access tenants within their assigned properties; Compliance/Admin have org-wide read. Implement REST endpoints for /auth, /properties, /rooms, /assignments, /tenants, /documents, /reports, and /audit; enforce RBAC roles (ADMIN, OPS, SUPPORT, VIEWER), Zod validation, helmet, CORS, and rate limiting on login and upload routes; log create/update/delete/verify events in an AuditLog. Business rules: prevent overlapping Active tenancies beyond room capacity; mark tenants “Ready for tenancy” only when the configured mandatory documents are Verified; block deletion of occupied rooms or properties with dependents. In the web app, add login, a sidebar with Dashboard, Properties, Tenants, Documents, Reports, and Admin; a Properties list and detail (tabs: Rooms, Assignments, Activity); a Tenants list with filters; a 4-step Onboarding Wizard (Details → Assign Room → Upload Docs → Review) with inline Zod validation and client-side image compression; a Documents screen to upload/preview and allow authorized users to Verify/Reject with reasons; simple Reports for occupancy by property, tenants missing required docs, and caseload per worker; plus CSV export and guarded routes by role. Keep costs low: skip email/SMS for MVP (in-app notifications only), use one Replit Deployment for prod with env secrets (DATABASE_URL, JWT keys, AES key, MAX_UPLOAD_MB), and enable Neon auto-sleep. Acceptance criteria: users can create properties, add rooms, assign support workers; onboard tenants into vacant rooms; upload and retrieve encrypted documents via signed, expiring tokens; verify/reject documents; run search/filters and CSV exports; RBAC is enforced everywhere; and the system runs reliably on Replit/Neon without paid object storage.
